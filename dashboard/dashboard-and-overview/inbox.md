# Inbox

## Summary

The **Inbox** feature in the nFinia digital banking platform serves as the central communication hub for business banking users. It consolidates platform notifications and support interactions into a single, easy-to-navigate interface. Through the Inbox, users can view system-generated notifications and manage support tickets — enabling seamless, traceable communication between business users and the Tyfone/Summerville support team.

The Inbox is divided into two primary sections:

* **Notifications** — displays system alerts, transaction updates, and platform messages
* **Support Messages** — a full-featured ticketing system for submitting, tracking, and managing support requests

***

## Use Cases

### 1. Monitoring Platform Notifications

Business users can stay up to date with real-time alerts about account activity, transaction statuses, approvals, and other system-generated events without leaving the digital banking platform.

### 2. Submitting a Support Request

When users encounter issues or need assistance (e.g., questions about ACH transfers, wire requests, card orders, or general account inquiries), they can submit a new support ticket directly from the Inbox — without needing to call or email externally.

### 3. Tracking Open Support Tickets

Users and administrators can view all existing tickets, their current status (Open, Closed, Pending), the last updated timestamp, and the assigned support team — providing full visibility into outstanding issues.

### 4. Attaching Supporting Documents to Tickets

When submitting a ticket, users can attach files (PDF, DOC, XLS, CSV, PNG, JPG, etc.) to provide additional context, such as transaction records, forms, or screenshots, improving resolution speed.

### 5. Cross-Team Ticket Assignment Visibility

The Support Messages view shows which Tyfone team member or group a ticket is assigned to, enabling transparency and accountability across support workflows.

## End-to-End Workflow

### Step 1 — Click on inbox widget on the top right of the screen

Click on the inbox widget on the top right of the screen next to log out

### Step 2 — Open the Inbox

From any screen in the nFinia platform, click **Inbox** in the left-hand navigation menu. This loads the main Inbox view, which defaults to the **Notifications** tab. Notifications include account-level alerts, transaction confirmations, and system messages relevant to the logged-in user.

![Step 1 – Inbox Main Screen](<../../.gitbook/assets/Unknown image>)

***

### Step 3 — Navigate to Support Messages

Click the **Support Messages** tab within the Inbox. This opens the support ticketing dashboard, which displays a paginated list of all tickets associated with the business account.

Each row in the table includes:

* **ID** — unique ticket identifier
* **Subject** — brief description of the inquiry (e.g., _Address Change Form_, _ACH Origination Form_, _Wire – Domestic Outgoing Wire Transfer Request_)
* **Last Updated** — relative timestamp of the most recent activity
* **Assigned To** — the Tyfone team or individual responsible for the ticket
* **Status** — current state: `Open`, `Closed`, or `Pending`

Users can filter tickets using the **View** dropdown (e.g., All Tickets) and search by keyword using the search bar.

![Step 2 – Support Messages Tab](<../../.gitbook/assets/Unknown image (1)>)

***

### Step 4 — Create a New Ticket

Click the **New Ticket** button located in the top-left of the Support Messages view. A modal dialog titled **"Create a new ticket"** appears with the following fields:

| Field            | Description                                                                                            |
| ---------------- | ------------------------------------------------------------------------------------------------------ |
| **Inquiry Type** | Dropdown to categorize the request (e.g., _General Inquiry_)                                           |
| **Subject**      | Text field for a brief description of the issue                                                        |
| **Message**      | Free-text area for detailed explanation of the request                                                 |
| **Choose Files** | Optional file attachment (max 20 MB; supported formats: txt, doc, docx, pdf, jpg, csv, xlsx, gif, tif) |

After filling in all required fields and optionally attaching a file, click **Send Message** to submit the ticket.

![Step 3 – Create a New Ticket Modal](<../../.gitbook/assets/Unknown image (2)>)

***

### Step 5 — Ticket Confirmation

Upon successful submission, a **Success Message** banner appears at the top of the Inbox screen confirming:

> _"Your ticket has been created successfully and your ticket ID is (JTTSE)."_

The newly created ticket is now visible in the Support Messages list and will be tracked until resolution.

![Step 4 – Ticket Submission Confirmation](<../../.gitbook/assets/Unknown image (3)>)
