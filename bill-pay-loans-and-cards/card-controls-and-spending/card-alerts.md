# Card Alerts

_Module: Banking › Cards › Card Details › Card Controls › Card Alerts_

## Summary

Card Alerts gives members direct control over how, where, and when their card can be used. From the Card Alerts tab inside Card Controls, members can configure spend thresholds that trigger real-time notifications and selectively enable or disable specific transaction types and merchant categories. Every change takes effect immediately — no branch visit, phone call, or waiting period required.

This level of granular control is particularly valuable for fraud prevention and budget management. A member can disable ATM withdrawals if they never use cash, block eCommerce purchases while keeping in-store use active, or receive an instant alert whenever a transaction exceeds a defined dollar amount. All controls are reversible at any time from the same screen.

## Key Use Cases


Set a spend alert threshold: Receive a notification whenever a single transaction exceeds a set dollar amount, enabling real-time awareness of large or unexpected charges.

Disable ATM withdrawals: Block the card from being used at ATMs entirely, useful for members who only use digital payments and want to reduce cash-related fraud risk.

Block eCommerce purchases: Prevent the card from being used for online transactions while keeping in-store functionality active — an effective control for shared household cards.

Restrict merchant categories: Block the card from being used at specific merchant types such as gas stations or restaurants, supporting budgeting goals or parental controls.

Enable only specific categories: Limit card acceptance to only the categories a member has explicitly approved, providing the tightest possible control over card spending.

## Step-by-Step Guide

_📍 Navigation path: Banking › Cards › [select card] › Card Details › Card Controls › Card Alerts tab_

### Step 1 — Navigate to Card Controls

From the nFinia top navigation, select **Cards**. Locate the card you want to configure and click **Card Controls** at the bottom of the card tile. This opens the Card Details page where all card management options are available.

<figure><img src="../../.gitbook/assets/Card_Alerts_img1.png" alt=""><figcaption></figcaption></figure>

### Step 2 — Select the Card Alerts Tab

Inside Card Details, find the Card Controls section and select the **Card Alerts** tab. The page displays two distinct sub-sections: **Spend Alerts** at the top, which controls notification thresholds, followed by **Transaction Type** and **Merchant Categories** controls that determine where and how the card can be used.

<figure><img src="../../.gitbook/assets/Card_Alerts_img2.png" alt=""><figcaption></figcaption></figure>

### Step 3 — Configure Spend Alerts

In the Spend Alerts section, you will see a **Per Transaction** toggle. Turn it on to activate spend threshold alerts, then click **Edit** to enter the maximum dollar amount for a single transaction. Any transaction that exceeds this amount will immediately trigger an alert notification to your enrolled device. This is one of the most effective tools for catching unauthorized charges as they happen rather than discovering them later on a statement.

> ℹ️ **Tip:** Members can set separate spend alert thresholds for different card types. Each card displays its own Per Transaction control, so a business card and a personal card can have different threshold settings.

### Step 4 — Configure Transaction Type Controls

The Transaction Type section lets you enable or disable specific ways your card can be used. Toggling any type to **Off** blocks that transaction method immediately for the card — no confirmation screen or delay.

| Transaction Type | Description |
| ------------------ | --------------------------------------------------- |
| In Store | Physical card-present purchases at retail locations |
| Mail / Phone Order | Orders placed by phone or mail using card details |
| ATM | Cash withdrawals and balance inquiries at ATMs |
| eCommerce | Online purchases and card-not-present transactions |
| Other | All other transaction types not listed above |
| Grocery | Purchases at grocery stores and supermarkets |

### Step 5 — Configure Merchant Category Controls

| Merchant Category | Description |
| ----------------- | ---------------------------------------------------------------- |
| Department Store | Purchases at department and general merchandise stores |
| Gas Station | Fuel and convenience store purchases at filling stations |
| Household | Home improvement, utilities, and household supply stores |
| Restaurant | Dining, cafes, bars, and food delivery services |
| Age Restricted | Merchants that sell age-restricted goods such as tobacco or alcohol |
| Personal Care | Salons, spas, pharmacies, and personal care service providers |

> ⚠️ **Note:** Disabling a Transaction Type or Merchant Category blocks the card for that control across all transactions on the account. Changes are applied instantly — ensure the restriction is intentional before toggling a category Off, as it will affect all pending and future transactions of that type.
