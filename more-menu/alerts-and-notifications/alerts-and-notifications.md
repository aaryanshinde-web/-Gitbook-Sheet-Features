# Alerts & Notifications

> **Module:** Banking › Settings → Alerts |

## Summary

The Alerts & Notifications module allows members to configure real-time account alerts delivered via push notification, SMS, or email. Members receive immediate notifications for account events that matter to them — including low balance warnings, large transactions, successful transfers, payment postings, security events, and new eDocument availability — without needing to log in to check manually.

Members can set general alerts that apply system-wide, such as login notifications and security events, as well as account-specific alerts configured per individual account, such as balance threshold warnings and large transaction triggers. The Do Not Disturb schedule mutes non-critical notifications during specified quiet hours while maintaining delivery of critical security alerts at all times, ensuring members are never inconvenienced by late-night notifications but are always protected.

Alert history is maintained in the Inbox, giving members a complete record of all alerts they have received. The Scheduled/Recurring Transfer Alerts feature notifies members before and after scheduled transfers are executed, providing full visibility into automated payment activity.

**At a Glance**

| Attribute | Detail |
| ----------------- | -------------------------------------------------------------------------- |
| Module | Settings > Alerts |
| Alert Types | Balance, Transaction, Security, Login, Payment Posted, eDocument Available |
| Delivery Channels | Push Notification, SMS, Email |
| Account-Specific | Balance threshold and large transaction alerts configurable per account |
| Do Not Disturb | Time-window to mute non-critical alerts during quiet hours |

## Key Use Cases


Members monitoring overdraft risk set a balance alert threshold per account so the system notifies when the balance drops below a defined amount, early warning before an overdraft occurs, giving the member time to transfer funds.

Members detecting fraud configure an alert for any charge above a set dollar amount, immediate notification of unexpected large transactions, enabling rapid response to unauthorized activity.

Security-conscious members enable a login alert so a notification is sent for every new sign-in event, real-time detection of unauthorized login attempts, even when the member is not actively using the app.

Members avoiding overnight alerts set a DND time window to mute non-critical notifications during sleep hours, reduces sleep disruption while ensuring that critical security alerts are always delivered regardless of the DND setting.

## Step-by-Step Guide

\| _Navigation: Dashboard > Settings (gear icon) > 'Alerts' OR More > Settings > Alerts._ |

**Step 1 — Start from Dashboard**

After logging in, you land on the Dashboard, which displays all account balances, upcoming payments, quick-action tiles, and the top navigation bar. This is the starting point for reaching Alert Settings, which can be accessed through the Settings gear icon in the navigation bar or through the More menu.

<figure><img src="../../.gitbook/assets/img_eb0a7d21423c.png" alt="" width="480"><figcaption></figcaption></figure>

**Step 2 — Open the More Menu**

Click **More** in the top navigation bar to expand the More options panel, which lists additional features and settings. From this panel, select **Alert Settings** to navigate directly to the alerts configuration page.

<figure><img src="../../.gitbook/assets/img_ba578ecd81d3.png" alt="" width="620"><figcaption></figcaption></figure>

**Step 3 — Navigate to Alert Settings**

The Alert Settings page loads with four tabs: **General Alerts**, **Account-Specific Alerts**, **Alert Settings**, and **Do-Not-Disturb**. The General Alerts tab is shown by default and displays all configurable alert categories across the platform. Use the tabs to navigate between different types of alert configuration.

<figure><img src="../../.gitbook/assets/img_e9ea763cf298.png" alt="" width="480"><figcaption></figcaption></figure>

**Step 4 — Configure Alert Delivery Channels**

Select the **Alert Settings** tab to manage how alerts are delivered to your device. This tab displays toggles for enabling or disabling each delivery channel independently: push notifications, SMS alerts, and email alerts. Your registered phone number and email address are shown beneath each channel toggle so you can confirm which contact details will receive alerts before saving your preferences.

<figure><img src="../../.gitbook/assets/img_9ad5af02d744.png" alt="" width="480"><figcaption></figcaption></figure>

**Step 5 — Configure Account-Specific Alert Settings**

Select the **Account-Specific Alerts** tab to set up balance and transaction threshold alerts at the individual account level. This tab lists all account-level alert types, such as Balance Alerts and Deposit Alerts. Clicking **Add a new alert** opens a modal dialog where you select the account, choose the alert condition (for example, balance falling below a certain amount), and set the threshold value. These targeted alerts are especially useful for members monitoring accounts that are at risk of overdraft or that receive irregular large deposits.

<figure><img src="../../.gitbook/assets/img_22574ef20eff.png" alt="" width="480"><figcaption></figcaption></figure>

**Step 6 — Set Do Not Disturb Hours**

Select the **Do-Not-Disturb** tab to configure a quiet period during which non-critical alert notifications will be suppressed. Enter the start time and end time for your quiet window. All optional and general account alerts are held during this period, while mandatory security alerts — such as login notifications and password changes — continue to be delivered immediately. This ensures you are never caught off guard by a security event, even when you have muted routine notifications.
